Manifesto

Instagram DM運用を、SaaSの都合から引き戻す。

ベンダーの管理画面に合わせて運用を曲げるのではなく、自社の運用に合わせてプロダクトを持つ。 GramStep はそのための土台です。安さだけではなく、コードとデータの主導権を取り戻すことが目的です。

Comparison

なぜ GramStep なのか

一般的なSaaS型DMツールで不足しやすい「コード公開」「データ所有」「API拡張」を、 最初から自社側に残せる構成です。

項目 一般的なDM SaaS GramStep
ソースコード 非公開 MIT
データ保管 ベンダー依存 自社Cloudflare環境
初期構築 速い やや重い
運用自由度 中 高
API拡張 制限されがち 自前で追加可能
自動化ロジック UI依存 Worker側で拡張可能

Problem

Instagram DM運用、こうなりがちです

ツール依存が強い

データ構造も配信ロジックもSaaS都合に閉じていて、細かい運用改善が重い。

Meta設定が属人化しやすい

Webhook、OAuth、App Review のどこで何を入れるべきかが散らばりやすい。

AI連携しにくい

非公開APIや制限の強いUI中心設計だと、Claude Code や自前の自動化に繋ぎづらい。

Features

管理画面だけで終わらない、運用基盤としての機能

DM自動返信

キーワード、全DM、Ice Breaker、コメント起点で自動応答。

シナリオ / ステップ配信

時間差配信、条件分岐、タグ付け、スコア更新をまとめて実行。

CRM / セグメント

タグ、スコア、プロフィール、メタデータをベースに配信対象を整理。

1:1チャット

HUMAN_AGENT ウィンドウ内で手動返信。オペレーション画面も統合。

配信分析

送信数、既読率、クリック率、CV、アカウントヘルスを追跡。

CLI セットアップ

Cloudflare リソース作成、Secrets設定、Meta設定導線を段階的に案内。

Setup

1

Cloudflare リソース作成

2

Worker / Admin をデプロイ

Meta 設定を進める

本番運用へ寄せる

FAQ

よくある質問

本当に無料ですか？

GramStep 自体は MIT ライセンスです。Cloudflare 側の使用量によっては有料化が必要ですが、それは Cloudflare への支払いです。

エンジニアでなくても運用できますか？

管理画面運用は可能ですが、初期セットアップと Meta 設定は技術理解があるチームの方が安全です。

既存のSaaSから移行できますか？

できます。ただし配信ロジック、タグ設計、審査フロー、Meta 権限まわりは自社側で再整理が必要です。

どこから始めればいいですか？

まずは GitHub の README と Quick Start を読み、Cloudflare と Meta のアカウントを準備してください。
